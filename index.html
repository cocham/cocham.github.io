Playlist Registration Flow
Spotify 링크 → PLYY 플레이리스트 생성
1
요청 수신 & 검증
POST /api/v1/playlists — 링크·코멘트·태그·커버 이미지 포함. Curator·MasterGenre DB 조회 및 검증.
2
토큰 획득 (Tiered Cache)
L1 → L2 → Spotify Auth API 순서로 서버 토큰 조회. AtomicBoolean으로 중복 발급 차단.
3
외부 메타데이터 수집
SpotifyPlaylistAdapter → GET /playlists/{id}. 트랙 목록·썸네일·앨범 정보 파싱 후 ExternalPlaylistData DTO 반환.
4
TrackSynchronizer — 중복 방지
spotify_id IN(...) Bulk 조회로 N+1 방지. DB에 없는 트랙만 saveAll()로 Batch Insert.
5
PlaylistFactory — 도메인 객체 생성
트랙 순서·큐레이션 코멘트·장르·무드·태그 연관관계 캡슐화. @Transactional 범위 내 단일 save.
6
201 Created 응답
Location: /api/v1/playlists/{id} 반환.
Tag Classification Strategy
입력된 태그명을 Genre → Mood → Custom 우선순위로 자동 분류합니다.
"힙합" → 장르/서브장르 DB 포함? Yes → GENRE
"새벽감성" → 무드 DB 포함? Yes → MOOD
"털ㄴ업" → 어디에도 미해당 → CUSTOM
